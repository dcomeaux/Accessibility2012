Accessibility Summary Report

Bobby Approval

Congratulations! These web pages contain no accessibility errors that Bobby can detect. There are, however, some checkpoints that an automatic program like Bobby cannot examine. These item(s) are presented below. You must be able to respond to these questions affirmatively in order to obtain Bobby Approval. If this summary report includes all the pages on your web site, you are entitled to use the Bobby Approved icon on your web site. A copy of the image (approved.gif) can be found in the images folder of the Bobby installation folder.

High priority items, manual check

Although these pages pass Bobby's criteria on items that it can check, there are other items that must be addressed in order to meet the minimum standard for accessibility as defined by the W3C. These 6 item(s) are presented below.

  1. Ensure that descriptions of dynamic content are updated with changes in content.
  2. If this table contains data in rows and columns (i.e. a spreadsheet), have you identified headers for the table rows and columns?
  3. Provide alternative content for each SCRIPT that conveys important information or function.
  4. If any of the images on this page convey important information beyond what is in each images alternative text, add descriptive (D) links.
  5. If any of the images on this page convey important information beyond what is in each images alternative text, add a LONGDESC attribute.
  6. Make sure that style sheets transform gracefully.


Section 1: Severity of accessibility errors and issues

This report ranks accessibility errors and recommendations based on their severity. Severity can be either High Priority, Medium Priority, or Low Priority. Most, but not all, High Priority items affect the Bobby Approval rating.
  • [High Priority] Make sure that style sheets transform gracefully.
  • [High Priority] Provide alternative content for each SCRIPT that conveys important information or function.
  • [High Priority] If any of the images on this page convey important information beyond what is in each images alternative text, add descriptive (D) links.
  • [High Priority] If any of the images on this page convey important information beyond what is in each images alternative text, add a LONGDESC attribute.
  • [High Priority] If this table contains data in rows and columns (i.e. a spreadsheet), have you identified headers for the table rows and columns?
  • [High Priority] Ensure that descriptions of dynamic content are updated with changes in content.
  • [Medium Priority] Do not use pop-up windows or change active window unless the user is aware this is happening.
  • [Medium Priority] Have you provided a linear text alternative for all tables that lay out content in parallel, word-wrapped columns?
  • [Medium Priority] Only use list elements for actual lists, not formatting.
  • [Medium Priority] Make sure that headings are nested properly.
  • [Medium Priority] Did you avoid using movement where possible?
  • [Medium Priority] Do labels of all form controls immediately follow its control on the same line?
  • [Medium Priority] Adjacent links should be separated.
  • [Medium Priority] Create link phrases that make sense when read out of context.
  • [Medium Priority] Style sheets should be used to control layout and presentation wherever possible.
  • [Medium Priority] Associate labels with their form controls.
  • [Medium Priority] Use relative sizing and positioning (% values) rather than absolute (pixels).
  • [Medium Priority] Have you grouped related form controls and labeled each group?
  • [Medium Priority] Mark up quotations with the Q and BLOCKQUOTE elements.
  • [Low Priority] Identify the language of the text, and any changes in the language.
  • [Low Priority] Consider adding keyboard shortcuts to frequently used links.
  • [Low Priority] Furnish keyboard shortcuts for form elements.
  • [Low Priority] Provide abbreviations for lengthy row or column labels.
  • [Low Priority] Specify a logical tab order among form controls, links and objects.
  • [Low Priority] If this table is used to display data in rows and columns (i.e. a spreadsheet), have you provided a summary of the table?
  • [Low Priority] Use the ABBR and ACRONYM elements to denote and expand abbreviations and acronyms.
